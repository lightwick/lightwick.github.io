# Intuition
The two possible operations are *splitting*(the first given operation) and *merging*(the second given operation). **(given that the requirements are satisfied)**
The two operations are the complete opposite of each other, meaning every action is reversible.\
This means if $a$ can be turned into $b$, then $b$ can also be turned into $a$.

From this we can conclude that if $a$ can be turned into $b$, then the completely *splitted* version of $a$ and $b$, i.e. the results of continuously *splitting* $a$ and $b$ until there is nothing more to *split*, would have to be the same.

# Solution & Implementation
## Solution 1 - memory limit exceeded
We can create a `vector` for the array inputs($a$, $b$).\
Now, as we take inputs, if we can *split* the element of the given array, we *split* the element and put it into the newly created vector recursively.\
This method is highly intuitive and easy to understand, but is highly inefficient as the final resulting `vector` could have a very large size.\
In the codeforces judge program, it was given a verdict of **memory limit exceeded**.
```
using namespace std;
 
int m;
 
void get(vector<int>& vec, int n) {
    if(n%m!=0) {
        vec.push_back(n);
    }
    else {
        for (int i = 0; i < m; i++) {
            get(vec, n/m);
        }
    }
}
 
void getInput(vector<int>& vec, int n) {
    for (int i = 0; i < n;i++) {
        int input;
        cin >> input;
        get(vec, input);
    }
}
 
void solve()
{
    int n;
    cin >> n >> m;
 
    vector<int> v1, v2;
 
    getInput(v1, n);
    int k;
    cin >> k;
    getInput(v2, k);
 
    if(v1==v2)
        cout << "Yes\n";
    else
        cout << "No\n";
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    while (t--)
    {
        solve();
    }
 
    return 0;
}
```
## Solution 2 - Accepted
Instead of reconstructing the given array, we keep track of the number of elements one element of the given array can be *split* into.\
This is implemented by a `vector` of `pair`s where the first element is the actual number of the *splitted* array and the second element is the number count of the first element of the `pair`.\
To put it simply, if we were to go through the vector in order, put the first element of the `pair` repeatedly by the second element, we would get the completely *splitted* array.

If the two *splitted* versions of the given arrays match, then it is possible to turn $a$ into $b$;
```
int m;
 
void getInput(vector<pair<int,long long>>& vec, int n) {
    for (int i = 0; i < n;i++) {
        int input;
        cin >> input;
        long long count = 1;
 
        while (input%m==0) {
            input /= m;
            count*=m;
        }
 
        if(!vec.empty() && vec.back().first == input)
            vec.back().second += count;
        else
            vec.push_back({input, count});
        
    }
}
 
void solve()
{
    int n;
    cin >> n >> m;
 
    vector<pair<int,long long>> v1, v2;
 
    getInput(v1, n);
    int k;
    cin >> k;
    getInput(v2, k);
 
    if(v1==v2)
        cout << "Yes\n";
    else
        cout << "No\n";
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    while (t--)
    {
        solve();
    }
 
    return 0;
}
```
